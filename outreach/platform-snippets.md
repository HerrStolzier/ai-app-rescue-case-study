## Upwork Proposal Snippet

I work best on small, concrete failures in existing apps: one broken route, Supabase write, API issue, or automation script. I built a public demo that shows my process end to end: failure, diagnosis, minimal fix, verification, and handover. If you can share the repo or a minimal reproduction, I can keep the scope tight and leave you with a verified patch plus a short technical handover.

## Fiverr Service Copy

I will fix one focused issue in your existing Next.js, Vercel, Supabase, API, or Python automation workflow. Good fits are broken API routes, failed deploy paths, Supabase auth/RLS write issues, webhook/API bugs, and small report automation scripts. Delivery includes the code change, verification steps, and a short technical handover.

## Contra Service Copy

Focused app rescue for founders and small teams: I diagnose one broken workflow in an existing app, apply a minimal fix, verify it, and document what changed. Best fit: Next.js/Vercel, Supabase auth/RLS, API integrations, Python reporting scripts, and AI-generated app cleanup.

## Direct Reply

I can help if this is a focused fix rather than a full rebuild. The way I work is: reproduce the failure, identify the root cause, make the smallest useful patch, run verification, and send a short handover. I have a public demo repo showing that process across Next.js/Vercel, Supabase, Python automation, and AI-built app hardening.

## Scope Guard

Good fit:

- one repo or minimal reproduction
- one broken route, API, auth/write path, deploy problem, or automation workflow
- clear expected vs actual behavior
- a setup/build/test command

Not a fit:

- full app rebuilds
- secrets in chat
- production data before an active contract
- vague "make this work" scopes without reproduction steps
